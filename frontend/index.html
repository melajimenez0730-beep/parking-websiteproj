Core Features

Everything you need to manage parking

From real-time spot visibility to distributed data across 3 shards, ParkSmart handles it all.

🗺️
Interactive Parking Map

Visual 3×4 grid layout per floor with live status updates. Hover spots to see vehicle info and availability.

🧠
Smart Spot Selection

Algorithm-driven recommendations: Nearest to Entrance, Exit, Grocery, or Disability-accessible spots.

⚡
Optimistic Concurrency

Version-based OCC prevents double-booking. Conflicting writes are detected and rejected at the database level.

🔒
3-Minute Soft Lock

Spots are held for 3 minutes during checkout. Expired locks auto-release so other users can book immediately.

🗄️
Floor-Level Sharding

MongoDB zone sharding partitions data by floor_number. Each shard holds exactly one floor's data.

📊
Staff Dashboard

Role-authenticated dashboard with cross-shard transaction log, floor-level stats, and real-time occupancy overview.

System Architecture

How the distributed cluster works

A mongos query router sits in front of three zone-sharded replica sets, routing every query to exactly the right floor-level shard.

Query Router
Mongos
Receives all queries from the API. Uses the shard key (floor_number) to route single-floor reads to one shard and fan-out cross-floor aggregations.

User Flow

Reserve a spot in 4 steps

Choose a Floor

Select Level 1, 2, or 3. The system fetches live availability from the corresponding shard.

Select a Spot

Browse the interactive grid or use smart-sort to find the closest spot to your destination.

Soft Lock (3 min)

Your spot is held with an OCC-protected soft lock while you complete your vehicle details.

Confirm & Park

Submit your booking. The system atomically converts the soft lock to a confirmed reservation.
